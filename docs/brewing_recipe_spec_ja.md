# Smart8 抽出レシピ仕様書

この文書は、バリスタが Smart8 / HARIO Smart7 EVS-70 向けの抽出レシピを検討するための仕様メモです。  
Smart8 の実装と、添付解析データ由来の通信仕様をもとにしています。

## 目的

Smart8 で扱う「抽出レシピ」は、Smart7 本体へ送信できる以下の情報に整理されます。

- レシピ名
- 湯温
- 粉量
- 複数の注湯工程
- 各工程の湯量、注湯時間、待機時間

このうち、本体へ直接送信されるのは「湯温」と「各工程」です。  
粉量は本体制御用の値ではなく、レシピ設計・表示・再現性のためのメタ情報です。

## レシピ項目

| 項目 | 意味 | Smart8 の編集範囲 | 本体送信 |
| --- | --- | --- | --- |
| レシピ名 | ユーザーが識別する名前 | 任意文字列。空欄時は「マイレシピ」 | 送信しない |
| 湯温 | 本体へ設定する温度 | 80〜96 ℃ | 送信する |
| 粉量 | 使用するコーヒー粉量 | 1〜60 g | 送信しない |
| 総湯量 | 各工程の湯量合計 | 自動計算 | 直接は送信しない |
| 工程数 | 注湯工程の数 | 1〜8 工程 | 各工程を送信 |
| 工程ごとの湯量 | その工程で注ぐ湯量 | 10〜500 ml、10 ml刻み | 送信する |
| 工程ごとの注湯時間 | その工程で湯を出す時間 | 0〜255 秒 | 送信する |
| 工程ごとの待機時間 | 注湯後、次工程まで待つ時間 | 0〜255 秒 | 送信する |

## 工程の意味

1工程は以下の3値で構成されます。

```text
湯量 ml / 注湯時間 秒 / 待機時間 秒
```

例:

```text
40 ml / 12 秒 / 45 秒
```

この例では、40 ml を12秒かけて注ぎ、その後45秒待機して次工程へ進む想定です。

最後の工程の待機時間は通常 `0秒` にします。  
最後に待機時間を入れても本体側で意味を持つ可能性はありますが、現時点では味づくり上の明確な用途を確認していません。

## 送信順序

Smart8 は抽出開始時に、以下の順序で本体へ送信します。

1. レシピ初期化
2. 工程1
3. 工程2
4. 工程3...
5. 湯温設定
6. 抽出開始

送信間隔はアプリ内部で以下のように固定しています。

| 送信内容 | 前の送信からの待ち時間 |
| --- | --- |
| レシピ初期化 | 0 ms |
| 各工程 | 200 ms |
| 湯温設定 | 100 ms |
| 抽出開始 | 100 ms |

この送信間隔は Bluetooth 通信を安定させるためのアプリ側の待ち時間です。  
味づくり上の「待機時間」ではありません。

## 本体へ送る工程データ

各工程は、内部的には以下の値として送信されます。

```text
工程番号, 湯量 / 10, 注湯時間, 待機時間
```

例:

```text
工程1: 40 ml / 12秒 / 45秒
送信値: 1, 4, 12, 45
```

湯量は10 ml単位で送るため、Smart8では10 ml刻みに丸めます。

## 抽出時間の考え方

レシピ上の目安抽出時間は、概算では以下です。

```text
合計注湯時間 + 合計待機時間
```

ただし、これは本体が実際に抽出完了するまでの厳密な時間ではありません。

実際には以下の影響を受ける可能性があります。

- 本体の加熱状態
- 注湯開始までの内部処理
- 流量補正や個体差
- フィルター、粉量、挽き目による透過速度
- 蒸らし中のドリッパー内滞留

そのため、バリスタ評価では「Smart8上の工程時間」と「実際の液抜け・味」を別々に確認してください。

## レシピ設計の検討観点

バリスタ側では、まず以下の軸で検討すると比較しやすいです。

### 1. 粉量と総湯量

Smart8では粉量は本体に送信されませんが、レシピの基準として重要です。

一般的な検討軸:

- 15 g / 240〜250 ml: 1杯用の標準
- 18 g / 270〜300 ml: 少し多めの1杯
- 20 g / 300〜330 ml: しっかりした量・濃度

### 2. 湯温

Smart8の編集範囲は80〜96℃です。

検討例:

- 浅煎り: 94〜96℃
- 中煎り: 90〜94℃
- 深煎り: 86〜90℃

ただし本体表示温度と抽出中の実湯温が完全一致するかは未検証です。

### 3. 蒸らし工程

最初の工程は蒸らしとして扱えます。

検討例:

- 湯量: 粉量の2〜3倍程度
- 注湯時間: 10〜15秒程度
- 待機時間: 30〜45秒程度

### 4. 分割注湯

現在の内蔵レシピは4工程を基本にしています。

検討例:

- 3工程: シンプルで短い抽出
- 4工程: 標準的で調整しやすい
- 5工程以上: 味の分離や濃度調整を細かく見たい場合

Smart8のUIでは最大8工程まで編集できます。

### 5. 注湯時間

注湯時間は、機械が指定湯量を何秒かけて出すかの指定です。

短くすると:

- 注湯速度が速くなる想定
- 撹拌が強まりやすい可能性
- すっきりする場合も、荒くなる場合もある

長くすると:

- 注湯速度が遅くなる想定
- 抽出が穏やかになる可能性
- 甘さや丸さが出る場合も、抜けが悪くなる場合もある

実際の流量は本体の状態・補正・個体差の影響を受けるため、味見で確認してください。

### 6. 待機時間

待機時間は、次工程へ進む前に本体が注湯を止めて待つ時間です。

長くすると:

- 蒸らしや浸透時間が伸びる
- 成分が出やすくなる可能性
- 冷めや滞留の影響も出る

短くすると:

- テンポよく注湯する
- 全体時間が短くなる
- 透過が追いつかない場合はドリッパー内の水位が上がる可能性

## 確定している仕様

- 湯温は本体へ送信される
- 各工程の湯量、注湯時間、待機時間は本体へ送信される
- 粉量は本体へ送信されない
- 工程湯量は10 ml単位で送信される
- Smart8 UI上の工程数は最大8
- Smart8 UI上の湯温範囲は80〜96℃
- Smart8 UI上の粉量範囲は1〜60 g
- Smart8 UI上の工程湯量範囲は10〜500 ml
- Smart8 UI上の注湯時間・待機時間範囲は0〜255秒

## 未検証・注意事項

- 本体の実流量が指定値と完全一致するかは未検証
- 本体の実湯温が指定温度と完全一致するかは未検証
- 待機時間中に本体側で内部処理が入るかは未検証
- 最終工程の待機時間に実用上の意味があるかは未検証
- 粉量は表示用なので、本体側の抽出挙動には直接影響しない
- ここにある内蔵レシピは味の完成版ではなく、検討開始用のプリセット

## 技術補足

Smart7へ送る平文フレームは、先頭 `0x99`、コマンド、payload長、payload、checksum で構成されます。  
Bluetooth送信時には `0x54` から始まる包装形式に変換されます。

バリスタ検討では通常この情報は不要ですが、レシピデータとしては以下の変換が行われます。

```text
湯量 ml -> 湯量 / 10 の1バイト値
注湯時間 秒 -> 1バイト値
待機時間 秒 -> 1バイト値
```

例:

```text
70 ml / 21 秒 / 20 秒
=> 7, 21, 20
```

この制約により、湯量は10 ml刻みで考えるのが安全です。
